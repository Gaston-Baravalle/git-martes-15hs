Areas de Git

Working Directory

El area de trabajo hace referencia a la carpeta donde creamos el repositorio, muy rara vez utilizaremos la linea de comandos en esta zona, pero es donde realizaremos todas las modificaciones relacionadas con nuestro proyecto.

Staging Area (INDEX)

El area de control de cambios corresponde al archivo que se encuentra en la carpeta .git que se crea al iniciar un repositorio. Aqui se registran los archivos a los vamos a hacerle seguimiento y es el punto de comparacion para confirmar dichos cambios. Es necesario agregar los archivos con los que estemos trabajando a dicha area para avanzar con el historial de cambios, ya que git a utilizar el comando git add realiza una captura del codigo.

Repository (Local/Remote)

El repositorio corresponde a la base de datos donde se almacenan los cambios en formato BLOB(Binary Object File) y es el lugar al que podemos acceder cada vez que necesitemos revisar algun estado anterior de nuestro proyecto. Esto es posible gracias a los commits identificados median un numero de HASH.
